## 动态路由 addRoute 父路由名称 + 重导航修复
- **问题**：点击"基础设置"等动态路由页面显示空白，无导航栏布局
- **控制台报错**：`[Vue Router warn]: Parent route "/" not found when adding child route`
- **根因（双重）**：
  1. `router.addRoute('/', {...})` 第一个参数接受的是 **name**（`'default'`），不是 path（`'/'`）
  2. 更关键：**Vue Router 在 beforeEach 守卫中 addRoute 后不会自动使用新路由**，必须 `next({ ...to, replace: true })` 重新导航才能匹配
- **修复**：
  - `router.addRoute('/')` → `router.addRoute('default')`
  - addDynamicRoute 后 `next()` → `return next({ ...to, replace: true })`
- **教训**：
  - `addRoute(parentName, route)` 的 parentName 是 `route.name`
  - 导航守卫中动态添加路由后必须重新导航，否则当前导航不使用新路由

## 动态路由兜底机制（最终根治）
- **问题**：基础设置 OK（静态路由），但"基础设置 2"等新增菜单页面无布局
- **根因**：`pathComponentMap` 中没有新菜单的映射 → `getComponentPath()` 返回 null → 路由守卫跳过注册 → 命中通配符 NotFound（无 MainLayout 包裹）
- **最终修复**：路由守卫改为**兜底策略**
  - 有 `pathComponentMap` 映射 → 加载对应组件（原逻辑）
  - 无映射 → 用 `BuildingView` 兜底 + 注册到 `default` children 下（确保有布局）
  - 所有未匹配路径都会动态注册并重新导航
- **效果**：从菜单管理新增的任何页面都能正确显示导航布局（显示"页面开发中..."）
