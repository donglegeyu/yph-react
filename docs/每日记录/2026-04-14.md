## Tab 切换时二级菜单未展开 / 三级菜单未选中 Bug 修复

### 根本原因
SecondSidebar.vue 里有两个 `watch(activeFirstMenu, ...)` 互相干扰：
1. 第 216 行：`activeFirstMenu` 变化时无条件清空 `activeKey` 和 `expandedKeys`
2. 第 380 行：重复 watcher 同样清空 `activeKey`

当 Tab 切换触发 `syncMenuState` 正确设置了三个状态后，`activeFirstMenu` 同步变化，立刻触发 watcher 把 `activeKey` 和 `expandedKeys` 清空，导致菜单显示错误。

### 修复方案
1. **SecondSidebar.vue**：移除两个 `watch(activeFirstMenu)` 中清空 `activeKey/expandedKeys` 的逻辑
2. **FirstSidebar.vue**：在 `handleClick` 里手动点击一级菜单时主动清空 `appStore.activeKey` 和 `appStore.expandedKeys`，确保用户手动切换一级菜单时状态重置

### 架构原则
- Tab 切换同步状态：由 `syncMenuState` 原子性一次设置三个状态（`activeFirstMenu` + `activeKey` + `expandedKeys`）
- 用户点击一级菜单：由 `FirstSidebar.handleClick` 主动清空二级状态，再由用户点击二级菜单时重新设置
- 禁止使用 watcher 做状态清空，容易与其他写入路径冲突

## 更多面板提升一级菜单到导航栏

### 需求
从更多面板点击三级/一级菜单时，如果对应一级菜单不在导航栏显示区域，自动提升到导航栏最后一个位置（替换原来的最后一个），同时自定义导航栏排序跟随变动。

### 实现
1. **app store** 新增 `promoteToNav(firstKey)` 方法
   - 从 `businessMenus` 找到目标菜单
   - 如果已存在于 `customNavMenus`，跳过
   - 如果 `customNavMenus` 已满 6 个，移除最后一个
   - 追加目标菜单到末尾
   - 更新 `customNavMenus`（自动触发持久化到后端和 localStorage）

2. **MoreMenuDrawer.handleChildClick** — 点击三级菜单后调用 `promoteToNav`
3. **FirstSidebar.handleMoreSelect** — 点击更多面板一级菜单后调用 `promoteToNav`

### 数据流
`promoteToNav` → 更新 `customNavMenus` → `displayMenus` computed 自动取前 6 个 → 一级导航自动高亮 ✓
`promoteToNav` → `customNavMenus` watcher → `saveCustomNavMenus` API → 自定义导航栏排序同步 ✓
