## 修复的菜单路由问题

### 问题现象
- 点击"敏感词管理"等菜单时控制台报错：`No match found for location with path "/sensitive-words-list"`
- 一级菜单错乱（从"设置中心"变成"商品中心"）

### 根因分析
1. **SecondSidebar 的 routeMap** 配置了路由映射，但 router/index.ts 里没有对应的路由页面
2. **handleMenuClick** 函数点击菜单时没有动态查找并设置所属的一级菜单

### 修复方案
1. **routeMap 修复**：将不存在的路由改为 `/building` 占位页面
   - sensitive-words-list → /building
   - series-list → /building

2. **动态父级菜单查找**：修改 handleMenuClick，在点击任意三级菜单时自动查找并设置正确的一级菜单
   ```javascript
   // 从所有二级菜单中查找该菜单的父级
   for (const firstKey in allSecondMenus) {
     for (const secondMenu of secondMenus) {
       if (secondMenu.children?.some(c => c.key === key)) {
         foundParentKey = firstKey
         break
       }
     }
   }
   if (foundParentKey && foundParentKey !== activeFirstMenu.value) {
     selectFirstMenu(foundParentKey)
   }
   ```

3. **MainLayout 特殊处理**：跳转到 `/building` 时保持当前菜单状态不变
