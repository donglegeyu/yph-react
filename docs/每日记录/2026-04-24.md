## 问题复盘

### 问题 1：组件预览显示为一级菜单
**现象**：组件预览没有显示在正确的层级位置

**根因**：插入菜单时 `level` 字段设置为 `0`，应该是 `2`

**教训**：
- 新增菜单时必须正确设置 `level` 字段（一级=0，二级=1，三级=2）
- 三级菜单的 `icon` 字段应该为 `NULL`

**解决方案**：
```sql
UPDATE nav_menu SET level = 2 WHERE id = 77;
UPDATE nav_menu SET icon = NULL WHERE id = 77;
```

### 问题 2：页面样式不一致
**现象**：组件预览页面左侧间距、padding 与其他页面不同

**根因**：
- 使用了错误的 padding 值（24px vs 0.75rem）
- 使用了错误的圆角值（8px vs 0.38rem）
- 缺少父容器布局配置

**解决方案**：
1. ComponentPreview.vue 使用统一的样式值
2. MainLayout.vue 添加 router-view 的布局样式

### 问题 3：三级菜单显示图标
**现象**：组件预览左侧显示图标，与其他三级菜单不一致

**根因**：插入菜单时设置了 `icon='app'`，而所有三级菜单的图标都是 `NULL`

## 关键经验

### 菜单配置规范
1. **level 字段**：必须根据实际层级设置
   - 一级菜单：level = 0
   - 二级菜单：level = 1
   - 三级菜单：level = 2

2. **icon 字段**：
   - 一级/二级菜单：可设置图标
   - 三级菜单：必须为 NULL

3. **parent_id 字段**：必须指向正确的父菜单 ID

### 前端样式规范
- padding：`0.75rem`
- 圆角：`0.38rem`
- 颜色：使用 Ant Design 标准颜色
- 布局：使用 flexbox + overflow 控制
