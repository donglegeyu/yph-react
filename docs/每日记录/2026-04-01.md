# 2026-04-01 工作日志

## TabBar 选中态问题修复

### 问题
刷新页面后，首页和材料申请列表同时显示选中态。点击首页后，材料申请列表才退出选中态。

### 根因
- store 初始化时 `activeTabKey = 'material-apply-list'`
- 路由从 `/materials` 重定向到 `/home`
- `activateTabByPath('/home')` 直接 return，不处理 `activeTabKey`

### 修复
1. `activateTabByPath('/home')` 中清空 `activeTabKey`
2. `activeTabKey` 初始值改为空字符串 `''`
3. 空字符串表示首页选中态

### 文件修改
- `stores/app.ts`: `activateTabByPath` 函数 + `activeTabKey` 初始值

## 一级菜单图标修复

### 问题
新增的10个一级菜单使用了不存在的图标名称，导致图标显示异常。

### 修复
从 iconpark-sprite.js 中查找可用的图标，为每个菜单配置正确的图标名称：

| 菜单 | 旧图标 | 新图标 |
|------|--------|--------|
| 会员中心 | user | user ✓ |
| 营销中心 | shoping | shopping-cart-add |
| 财务中心 | money | finance |
| 报表中心 | chart | chart ✓ |
| 设置中心 | setting | setting ✓ |
| 消息中心 | mail | mail ✓ |
| 数据中心 | database | internal-data |
| 权限中心 | shield | permissions |
| 运营中心 | operation | config |
| 质量中心 | check-circle | check-correct |
| 物流中心 | car | truck |

### 文件修改
- `stores/app.ts`: 更新 DEFAULT_FIRST_MENUS 图标名称
- `CACHE_VERSION`: 9 → 10（清除旧缓存）
- `public/iconpark-sprite.js`: 将 `stroke="#333"` 改为 `stroke="currentColor"`（使图标颜色跟随 CSS color 属性）

### 修复的图标
以下图标原来使用硬编码颜色，现在改为 currentColor：
- `finance` - 财务中心
- `internal-data` - 数据中心
- `truck` - 物流中心

还有其他所有使用 `#333` 的图标也都已修复。

## 图标检查工具

### 创建的文件
- `scripts/check-icons.js` - Node.js 图标检查脚本
- `scripts/check-icons.sh` - Shell 版本检查脚本
- `docs/ICON_SAFE_LIST.md` - 安全图标列表

### 使用方法
```bash
# 检查指定图标
node scripts/check-icons.js shopping-cart-add finance truck

# 检查所有硬编码颜色的图标
node scripts/check-icons.js

# 自动修复所有硬编码颜色
node scripts/check-icons.js --fix
```

### 添加新图标流程
1. 在 IconPark 选择图标并导出 SVG sprite
2. 运行检查脚本验证
3. 如有问题，使用 `--fix` 自动修复
4. 更新 `docs/ICON_SAFE_LIST.md`
