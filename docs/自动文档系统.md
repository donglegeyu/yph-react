# 自动文档系统使用说明

## 系统概述

本项目已配置完整的自动文档生成系统，包括：

- **后端 API 文档**: SpringDoc OpenAPI + Swagger UI
- **前端组件文档**: Storybook
- **开发手册**: 自动合并的完整文档
- **CI/CD 自动化**: GitHub Actions 自动更新

## 快速开始

### 1. 本地使用

#### 查看 Swagger API 文档
```bash
# 启动后端服务
cd material-system-server
mvn spring-boot:run

# 访问 Swagger UI
# 浏览器打开: http://localhost:8080/swagger-ui.html
```

#### 启动 Storybook
```bash
cd material-system
npm install
npm run storybook

# 访问: http://localhost:6006
```

#### 手动生成文档
```bash
# 生成组件文档
python3 scripts/generate_component_docs.py \
  --input material-system/src/components \
  --output docs/generated/components.json

# 生成 API 文档（需要后端运行）
# 或者使用示例文件
python3 scripts/generate_api_docs.py \
  --input docs/generated/api-docs.json \
  --output docs/API手册.md

# 合并完整开发手册
python3 scripts/merge_docs.py \
  --backend docs/API手册.md \
  --frontend docs/generated/components.json \
  --database docs/generated/database.md \
  --output docs/开发手册.md
```

### 2. CI/CD 自动更新

#### 触发方式
- **自动触发**: 推送代码到 `main` 或 `develop` 分支
- **手动触发**: 在 GitHub Actions 页面手动运行工作流
- **PR 触发**: 打开 PR 时会生成变更报告

#### 工作流文件
- `.github/workflows/auto-doc-update.yml`: 完整自动文档更新
- `.github/workflows/update-docs.yml`: 简化版文档同步
- `.github/workflows/auto-sync-docs.yml`: 文档同步（不自动提交）

## 为组件添加 Story

在 Vue 组件同级目录创建 `.stories.ts` 文件：

```typescript
// src/components/MyComponent.stories.ts
import type { Meta, StoryObj } from '@storybook/vue3'
import MyComponent from './MyComponent.vue'

const meta = {
  title: 'Components/MyComponent',
  component: MyComponent,
  tags: ['autodocs'],
  argTypes: {
    title: { control: 'text' },
  },
} satisfies Meta<typeof MyComponent>

export default meta
type Story = StoryObj<typeof meta>

export const Default: Story = {
  args: {
    title: '默认标题',
  },
}

export const WithLongTitle: Story = {
  args: {
    title: '这是一个很长很长的标题',
  },
}
```

## 为 Controller 添加 Swagger 注解

使用 SpringDoc 注解：

```java
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.tags.Tag;

@RestController
@RequestMapping("/api/materials")
@Tag(name = "材料管理", description = "材料相关接口")
public class MaterialController {

    @GetMapping
    @Operation(summary = "获取材料列表", description = "分页获取材料列表")
    public Result<PageResult<MaterialVO>> list(
        @RequestParam(required = false) String keyword,
        @RequestParam(defaultValue = "1") Integer pageNum,
        @RequestParam(defaultValue = "10") Integer pageSize
    ) {
        // ...
    }
}
```

## 注意事项

### ⚠️ 重要提示
1. **不要手动编辑自动生成的文件**
   - `docs/API手册.md`
   - `docs/开发手册.md`
   - `docs/generated/` 下的文件

2. **优先编辑源文件**
   - 组件文档: 编辑 `.stories.ts` 文件
   - API 文档: 添加 Swagger 注解
   - 其他文档: 编辑对应源文件

3. **提交前测试**
   - 确保本地能正常运行 Storybook
   - 确保 Swagger 注解正确

### 常见问题

**Q: Swagger 页面无法访问？**
A: 检查后端服务是否正常启动，访问 http://localhost:8080/swagger-ui.html

**Q: Storybook 组件不显示？**
A: 检查是否创建了对应的 `.stories.ts` 文件，文件格式是否正确

**Q: GitHub Actions 失败？**
A: 检查 Actions 日志，通常是依赖安装或构建问题
