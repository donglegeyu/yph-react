## 1. 产品概述

### 1.1 背景

当前系统缺乏权限控制机制，且已有的「域管理」页面为占位页。本方案将**域管理**与**权限系统**整合，构建一套完整的用户权限管理体系。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **域（Domain）** | 功能模块的组合，包含一组菜单 + 数据权限规则 |
| **菜单权限** | 用户可访问的功能菜单 |
| **数据权限** | 用户可查看的数据范围（全部/仅本人/自定义） |

### 1.3 设计原则

- **域 = 菜单集合 + 数据权限规则**：简化配置，批量分配
- **多对多关系**：一个用户可拥有多个域，一个域可分配给多个用户
- **按需查询**：支持按人查权限、按域查用户

## 2. 数据库设计

### 2.1 表结构

```sql
-- 用户表
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(255) NOT NULL COMMENT '密码（加密）',
    nickname VARCHAR(50) COMMENT '昵称',
    status TINYINT DEFAULT 1 COMMENT '1-启用 0-禁用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted TINYINT DEFAULT 0
);

-- 域表
CREATE TABLE sys_domain (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    domain_key VARCHAR(50) NOT NULL UNIQUE COMMENT '域标识',
    domain_name VARCHAR(100) NOT NULL COMMENT '域名称',
    description VARCHAR(255) COMMENT '描述',
    is_default TINYINT DEFAULT 0 COMMENT '1-默认域(不可调整层级排序) 0-自定义域',
    status TINYINT DEFAULT 1 COMMENT '1-启用 0-禁用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted TINYINT DEFAULT 0
);

-- 域-菜单关联表（支持域内个性化配置）
CREATE TABLE sys_domain_menu (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    domain_id BIGINT NOT NULL COMMENT '域ID',
    menu_id BIGINT NOT NULL COMMENT '菜单ID（关联nav_menu）',
    custom_label VARCHAR(100) COMMENT '域内自定义菜单名称（为空使用默认名称）',
    custom_level INT COMMENT '域内自定义层级（仅自定义域可用，为空继承系统层级）',
    custom_parent_id BIGINT COMMENT '域内自定义父菜单（仅自定义域可用）',
    sort INT DEFAULT 0 COMMENT '域内排序（默认域使用系统排序）',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_domain_menu (domain_id, menu_id)
);

-- 数据权限规则表（支持域级别和用户级别）
CREATE TABLE sys_data_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT COMMENT '用户ID（用户级别规则，可为空）',
    domain_id BIGINT COMMENT '域ID（域级别规则，可为空）',
    menu_key VARCHAR(50) NOT NULL COMMENT '菜单标识',
    filter_type VARCHAR(20) NOT NULL COMMENT '过滤类型：all/self/dept/custom',
    filter_field VARCHAR(50) COMMENT '过滤字段',
    filter_value VARCHAR(255) COMMENT '过滤值',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    -- 约束：user_id 和 domain_id 至少有一个
    CONSTRAINT chk_user_or_domain CHECK (user_id IS NOT NULL OR domain_id IS NOT NULL)
);

-- 用户-域关联表
CREATE TABLE sys_user_domain (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    domain_id BIGINT NOT NULL COMMENT '域ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_domain (user_id, domain_id)
);
```

## 3. 功能模块

### 3.1 域管理

**路由**：`/domain-manage`

| 功能 | 说明 |
|------|------|
| 域列表 | 分页展示所有域，支持状态筛选 |
| 新增域 | 选择菜单 + 配置数据权限 |
| 编辑域 | 修改域的菜单和数据权限 |
| 删除域 | 软删除，解除用户关联 |
| 分配用户 | 给域分配用户 |

**菜单来源说明**：
- 域只能选择「菜单管理」中已存在的菜单，不支持在域内新增菜单
- **同一菜单在不同域可配置不同的名称和层级**
- 菜单权限判断基于原始菜单 ID，与域内配置无关

**域类型说明**：

| 域类型 | 说明 | 域内菜单配置权限 |
|--------|------|-----------------|
| **默认域** | 「星际造梦」 | 仅支持重命名，不支持调整层级/排序 |
| **自定义域** | 其他自定义域 | 支持重命名、层级、排序、父菜单配置 |

**域内菜单配置说明**：
| 字段 | 默认域 | 自定义域 |
|------|--------|----------|
| 菜单名称 | 可修改 | 可修改 |
| 层级 | 固定 | 可调整 |
| 父菜单 | 固定 | 可调整 |
| 排序 | 固定 | 可调整 |

**新增/编辑域表单**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 域标识 | Input | 英文唯一标识 |
| 域名称 | Input | 中文名称 |
| 描述 | Textarea | 可选 |

**域内菜单配置**（选择菜单后配置）：

| 字段 | 类型 | 说明 |
|------|------|------|
| 菜单名称 | Input | 可自定义，覆盖系统默认名称 |
| 层级 | Select | 一级/二级/三级 |
| 父菜单 | TreeSelect | 可重新挂载 |
| 排序 | InputNumber | 域内排序 |

### 3.2 用户管理

**路由**：`/user-management`

| 功能 | 说明 |
|------|------|
| 用户列表 | 分页展示所有用户，支持状态筛选 |
| 新增用户 | 输入用户名、密码、昵称 |
| 编辑用户 | 修改用户信息 |
| 禁用/启用 | 修改用户状态 |
| 分配域 | 给用户分配域（多选） |
| 重置密码 | 管理员重置用户密码 |
| **数据权限** | **单独配置用户级别的数据权限规则（覆盖域规则）** |

### 3.3 权限查询与快速分配

**路由**：`/permission-query`

| 功能 | 说明 |
|------|------|
| 按人查权限 | 选择用户，查看其拥有的域、菜单、数据权限 |
| 按域查用户 | 选择域，查看分配了该域的所有用户 |
| **快速分配域** | **选择用户，直接增删其所属域** |
| **快速调整菜单** | **选择用户，直接增删其菜单权限** |
| **调整数据权限** | **选择用户，覆盖域级别的数据权限规则** |

## 4. 数据权限规则

### 4.1 规则类型

| 类型 | 值 | 说明 | 示例 |
|------|-----|------|------|
| **all** | 全部可见 | 可查看所有数据 | 无需配置 |
| **self** | 仅本人 | 仅查看自己创建的数据 | filter_field: creator_id |
| **dept** | 本部门 | 仅查看本部门数据 | filter_field: dept_id, filter_value: 用户所属部门 |
| **custom** | 自定义 | 自定义 SQL 过滤条件 | filter_field: dept_id, filter_value: 'engineering' |

### 4.2 规则层级

数据权限规则分为两级，**用户级别优先于域级别**：

| 级别 | 说明 |
|------|------|
| **域级别** | 定义域内用户的默认数据权限规则 |
| **用户级别** | 覆盖域级别，可为特定用户单独设置数据权限 |

### 4.3 规则配置示例

```json
// 域：材料管理员（域级别默认规则）
{
  "domainId": 1,
  "domainKey": "material-admin",
  "domainName": "材料管理员",
  "defaultRules": [
    {
      "menuKey": "material-apply",
      "filterType": "all",
      "filterField": null,
      "filterValue": null
    },
    {
      "menuKey": "purchase-demand",
      "filterType": "self",
      "filterField": "creator_id",
      "filterValue": null
    }
  ]
}

// 用户：公司A领导（用户级别，覆盖域规则）
{
  "userId": 100,
  "username": "leaderA",
  "overrideRules": [
    {
      "menuKey": "material-apply",
      "filterType": "custom",
      "filterField": "company_id",
      "filterValue": "companyA"
    }
  ]
}
```

### 4.4 规则生效逻辑

```
1. 查询用户权限时，先获取用户所属的域
2. 获取域级别的默认数据权限规则
3. 获取用户级别的覆盖规则
4. 合并规则：用户级别规则覆盖域级别规则
5. 最终应用合并后的规则
```

## 7. 业务流程

### 7.1 新用户入职

```
1. 管理员进入「用户管理」
2. 新增用户（用户名、密码）
3. 给用户分配域（选择已有的域）
4. （可选）单独配置用户级别数据权限，覆盖域规则
5. 用户登录后自动拥有该域的菜单和数据权限
```

### 7.2 创建新域

```
1. 管理员进入「域管理」
2. 新增域（域标识、名称）
3. 选择包含的菜单（从菜单树多选）
4. 配置数据权限规则（按菜单设置过滤类型）
5. 保存域
```

### 7.3 权限查询

```
按人查：
1. 进入「权限查询」页面
2. 选择「按人查询」
3. 输入或选择用户名
4. 查看：该用户的域列表、菜单权限、数据权限规则

按域查：
1. 进入「权限查询」页面
2. 选择「按域查询」
3. 选择域
4. 查看：该域下的所有用户列表

快速分配（在权限查询页）：
1. 选择用户
2. 点击「分配域」→ 增删用户所属域
3. 点击「调整菜单」→ 增删用户菜单权限
4. 点击「数据权限」→ 覆盖域级别的数据权限规则
```
