# 菜单状态管理规范

## 概述

本文档记录了菜单系统（二级菜单、三级菜单）的展开/收起状态管理规范，避免重复出现类似bug。

---

## 核心状态

### 1. 三个关键状态
- `activeFirstMenu` - 当前选中的一级菜单
- `activeKey` - 当前选中的菜单key（二级或三级）
- `expandedKeys` - 当前展开的二级菜单数组

### 2. 状态来源
- **Pinia Store**：`src/stores/app.ts`
- **二级菜单组件**：`src/components/layout/SecondSidebar.vue`
- **一级菜单组件**：`src/components/layout/FirstSidebar.vue`

---

## 已验证的场景和正确实现

### 场景一：用户点击三级菜单

**期望行为**：展开对应的二级菜单，选中当前三级菜单

**正确实现**：
```typescript
// SecondSidebar.vue - handleClick 函数
function handleClick(menu: any) {
  if (menu.path) {
    appStore.navigateToPath(menu.path)  // 设置 activeFirstMenu
    appStore.openTab(menu.key, menu.label, menu.path)
    activeKey.value = menu.key  // 设置 activeKey
    // 设置 expandedKeys
    if (menu.children?.length) {
      expandedKeys.value = [menu.key]
    } else {
      // 三级菜单：找到对应的二级父菜单并展开
      const parent = currentSecondMenus.value.find((m: any) =>
        m.children?.some((c: any) => c.key === menu.key)
      )
      if (parent) {
        expandedKeys.value = [parent.key]
      }
    }
    router.push(menu.path)
  }
}
```

### 场景二：Tab 切换

**期望行为**：同步三个状态，展开对应的二级菜单，选中对应菜单

**正确实现**：
```typescript
// app.ts - syncMenuState 函数
function syncMenuState(tabKey: string, tabPath: string) {
  for (const [firstKey, seconds] of Object.entries(secondMenusMap.value)) {
    for (const menu of seconds) {
      if (menu.path === tabPath || menu.key === tabKey) {
        activeFirstMenu.value = firstKey
        activeKey.value = menu.key
        expandedKeys.value = menu.children?.length ? [menu.key] : []
        promoteToNav(firstKey)
        return
      }
      if (menu.children?.length) {
        for (const sub of menu.children) {
          if (sub.path === tabPath || sub.key === tabKey) {
            activeFirstMenu.value = firstKey
            activeKey.value = sub.key
            expandedKeys.value = [menu.key]
            promoteToNav(firstKey)
            return
          }
        }
      }
    }
  }
  // 非菜单路径（如主题设置），清空状态
  // 注意：设置 activeFirstMenu = 'home' 而不是 ''，以便触发 FirstSidebar 的 watch
  activeFirstMenu.value = 'home'
  activeKey.value = ''
  expandedKeys.value = []
  secondSidebarHovered.value = false
}
```

### 场景三：路由变化（直接访问URL）

**期望行为**：自动识别菜单路径，同步展开对应菜单

**正确实现**：
```typescript
// SecondSidebar.vue - watch(route.path)
watch(() => route.path, (path) => {
  if (path === '/home' || path === '/') return

  const result = appStore.navigateToPath(path)
  if (result) {
    const menu = result.thirdMenu || result.secondMenu
    if (menu) {
      appStore.openTab(menu.key, menu.label, path)
    }
  } else {
    // 非菜单路径，清空状态
    activeKey.value = ''
    expandedKeys.value = []
  }
}, { immediate: true })
```

### 场景四：非菜单路径（主题设置等特殊页面）

**期望行为**：一级菜单不选中，二级菜单不选中，面板不显示

**正确实现**：
```typescript
// FirstSidebar.vue - handleThemeSettings
function handleThemeSettings() {
  adminPopoverVisible.value = false
  appStore.openTab('theme-settings', '主题设置', '/component-preview')
  // 立即清空状态
  appStore.activeFirstMenu = ''
  appStore.activeKey = ''
  appStore.expandedKeys = []
  appStore.secondSidebarHovered = false
  activeKey.value = ''
  router.push('/component-preview')
}
```

### 场景五：刷新页面

**期望行为**：
- 菜单页面：保持选中状态
- 非菜单页面（主题设置）：不显示选中状态

**正确实现**：

#### 5.1 app.ts - clearStateForNonMenuPath 函数
```typescript
// 检查并清空非菜单路径的状态
function clearStateForNonMenuPath() {
  const currentPath = window.location.pathname

  if (currentPath === '/home' || currentPath === '/') {
    return
  }

  // 检测当前路由是否在菜单配置中
  const isMenuPath = Object.values(secondMenusMap.value).some(menus =>
    menus.some(menu => menu.path === currentPath ||
      menu.children?.some(child => child.path === currentPath))
  )

  // 非菜单路径，清空状态
  if (!isMenuPath) {
    activeFirstMenu.value = ''
    activeKey.value = ''
    expandedKeys.value = []
    secondSidebarHovered.value = false
  }
}
```

#### 5.2 app.ts - fetchMenus 函数
```typescript
async function fetchMenus(forceRefresh = false) {
  if (!forceRefresh && menusLoaded && businessMenus.value.length > 0) {
    // 即使菜单已加载，仍检查非菜单路径
    clearStateForNonMenuPath()
    return
  }
  // ... API 请求逻辑 ...
  // 无论 API 成功还是失败，都检查非菜单路径状态
  clearStateForNonMenuPath()
}
```

#### 5.3 FirstSidebar.vue - watch(activeFirstMenu)
```typescript
// 监听 activeFirstMenu 变化（由 navigateToPath 更新时同步 activeKey）
watch(() => appStore.activeFirstMenu, (newFirstMenu) => {
  if (!isHovering.value) {
    if (newFirstMenu === 'home' || newFirstMenu === 'favorites' || !newFirstMenu) {
      // 非菜单路径，清空选中状态
      activeKey.value = ''
    } else {
      // 正常业务菜单
      activeKey.value = newFirstMenu
    }
  }
})
```

#### 5.4 FirstSidebar.vue - watch(route.path)
```typescript
watch(() => route.path, (path) => {
  if (path === '/home') {
    activeKey.value = 'home'
  } else if (path === '/favorites') {
    activeKey.value = 'favorites'
  } else {
    // 等待 secondMenusMap 加载完成后设置
    const validFirstMenuKeys = Object.keys(appStore.secondMenusMap)
    if (appStore.activeFirstMenu && validFirstMenuKeys.includes(appStore.activeFirstMenu)) {
      activeKey.value = appStore.activeFirstMenu
    }
  }
}, { immediate: true })
```

#### 5.5 SecondSidebar.vue - watch(secondMenusMap)
```typescript
// 监听 secondMenusMap 加载完成后恢复状态
watch(() => appStore.secondMenusMap, (newMap) => {
  if (Object.keys(newMap).length > 0) {
    if (route.path === '/home' || route.path === '/') {
      return
    }

    // 检测当前路由是否在菜单配置中
    const isMenuPath = Object.values(newMap).some(menus =>
      menus.some(menu => menu.path === route.path ||
        menu.children?.some(child => child.path === route.path))
    )

    // 非菜单路径，清空状态
    if (!isMenuPath) {
      appStore.activeFirstMenu = ''
      appStore.activeKey = ''
      appStore.expandedKeys = []
      activeKey.value = ''
      expandedKeys.value = []
      return
    }

    // 正常菜单路径，恢复一级菜单选中状态
    const result = appStore.navigateToPath(route.path)
    if (result) {
      // 恢复 activeKey
      const menu = result.thirdMenu || result.secondMenu
      if (menu) {
        activeKey.value = menu.key
      }
      // 恢复 expandedKeys
      if (result.secondMenu?.children?.length) {
        expandedKeys.value = [result.secondMenu.key]
      }
      // 强制触发 activeFirstMenu 变化，确保 FirstSidebar 也能更新
      // 当 activeFirstMenu 新旧值相同时，watch 不会触发，需要先清空再设置
      const currentFirstMenu = appStore.activeFirstMenu
      if (currentFirstMenu === result.firstKey) {
        appStore.activeFirstMenu = ''
        nextTick(() => {
          appStore.activeFirstMenu = result.firstKey
        })
      }
    }
  }
}, { immediate: true })
```

## 常见错误模式

### ❌ 错误1：watch 中无条件清空状态

**错误代码**：
```typescript
watch(() => appStore.activeFirstMenu, (newFirstMenu, oldFirstMenu) => {
  // 错误：清空了其他正常设置的状态
  activeKey.value = ''
  expandedKeys.value = []
})
```

**问题**：当 syncMenuState 正确设置三个状态后，activeFirstMenu 变化会触发这个 watcher，把其他两个状态清空

**正确做法**：根据条件判断是否清空或设置

---

### ❌ 错误2：条件过于保守导致状态无法更新

**错误代码**：
```typescript
if (keyToExpand && expandedKeys.value.length === 0) {
  expandedKeys.value = [keyToExpand]
}
```

**问题**：当之前已经展开了一个二级菜单时，这个条件不满足，导致新点击的菜单无法正确展开

**正确做法**：
```typescript
if (keyToExpand) {
  expandedKeys.value = [keyToExpand]
}
```

---

### ❌ 错误3：多个 watcher 互相干扰

**错误示例**：多个 watch(activeFirstMenu) 同时存在

**问题**：不同 watcher 可能执行顺序不确定，导致状态被意外覆盖

**正确做法**：保持单一数据源，使用 syncMenuState 原子性设置状态

---

### ❌ 错误4：非菜单路径未清空状态

**错误代码**：`syncMenuState` 在找不到匹配时直接 return，不清空状态

**问题**：切换到主题设置等非菜单页面时，之前的一级菜单仍然保持选中

**正确做法**：设置 `activeFirstMenu = 'home'` 以便触发 FirstSidebar 的 watch

---

### ❌ 错误5：watch 新旧值相同时不触发

**问题**：`watch(() => appStore.activeFirstMenu, ...)` 当设置的新值与旧值相同时不会触发

**解决策略**：
1. 先清空再设置：`activeFirstMenu = ''` → `nextTick(() => activeFirstMenu = 'xxx')`
2. 或者使用特殊标记值

## 架构原则

### 1. 状态设置原则

| 操作 | 设置状态 | 职责 |
|------|---------|------|
| Tab 切换 | `syncMenuState()` 原子设置三个状态 | app.ts |
| 用户点击菜单 | `handleClick()` 直接设置三个状态 | SecondSidebar.vue |
| 路由变化 | `navigateToPath()` + watch 组合 | app.ts + SecondSidebar.vue |
| 用户点击一级菜单 | `selectFirstMenu()` 只设置一级 | FirstSidebar.vue |
| 非菜单路径 | 所有状态清空 | app.ts + FirstSidebar.vue + SecondSidebar.vue |
| 刷新页面 | `clearStateForNonMenuPath()` 检测并清空 | app.ts |

### 2. FirstSidebar 的 activeKey 与 appStore.activeKey

**重要**：`FirstSidebar.vue` 中的 `activeKey` 是本地 ref，与 `appStore.activeKey` 是两个独立的变量。

- `appStore.activeKey` - 存储二级/三级菜单的选中状态
- `FirstSidebar.activeKey` - 用于显示一级菜单的选中高亮

两者通过 watch 同步：
- `FirstSidebar.activeKey = appStore.activeFirstMenu`（一级菜单 key）
- `SecondSidebar.activeKey = appStore.activeKey`（二级/三级菜单 key）

### 3. 禁止事项

- ❌ 禁止在 watch 中无条件清空 activeKey 或 expandedKeys
- ❌ 禁止设置过于保守的条件阻止状态更新
- ❌ 禁止多个 watcher 操作同一组状态
- ❌ 禁止在状态设置后又立即清空
- ❌ 禁止在 syncMenuState 返回时不清空非菜单路径状态
- ❌ 禁止忘记清空 FirstSidebar 本地的 activeKey

### 4. 调试技巧

打开 console.log 调试：
```typescript
console.log('[navigateToPath] path:', path,
            'secondMenusMap keys:', Object.keys(secondMenusMap.value),
            'current activeFirstMenu:', activeFirstMenu.value)
```
