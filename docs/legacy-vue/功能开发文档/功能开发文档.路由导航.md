## 二级菜单 (SecondSidebar)

- 支持 2 层菜单结构（父级 + children）
- 支持 3 层嵌套（三级菜单通过 `children` 属性定义）
- 选中色：`#F95914` / `#FFF7F0`
- 菜单定义在 `stores/app.ts` 的 `secondMenusMap`

### 二级菜单面板显示逻辑

**核心状态**：
| 状态 | 类型 | 说明 |
|------|------|------|
| `activeFirstMenu` | string | 当前选中的一级菜单 key |
| `activeKey` | string | 当前选中的二级/三级菜单 key |
| `expandedKeys` | string[] | 展开的二级菜单父级 key 数组 |
| `secondSidebarHovered` | boolean | 面板悬浮状态（鼠标操作） |
| `secondSidebarFixed` | boolean | 面板固定状态（用户切换） |
| `isFirstMenuHovering` | boolean | 鼠标是否悬停在一级菜单上 |

**显示条件**：
```
showSecondSidebar = secondSidebarHovered || secondSidebarFixed
```

**重要规则**：Tab 切换时不自动显示二级面板，只有鼠标实际悬停在一级菜单时才显示。

**交互场景说明**：
| 场景 | 一级菜单选中 | 二级菜单高亮 | 二级面板显示 |
|------|-------------|-------------|------------|
| 点击一级菜单 | ✅ | - | ✅ |
| 鼠标悬停一级菜单 | ✅ | - | ✅ |
| Tab 切换页面 | ✅ | ✅ | ❌ |
| 点击二级菜单 | ✅ | ✅ | ❌ |
| 点击固定按钮 | ✅ | ✅ | ✅ |
| 非菜单路径（如主题设置） | ❌ | ❌ | ❌ |

**状态流转**：
1. **Tab 切换** → `navigateToPath()` 更新 `activeFirstMenu` → 同步 `activeKey` 和 `expandedKeys` → 不显示面板
2. **鼠标悬停** → 设置 `isFirstMenuHovering = true` → `activeFirstMenu` 变化时显示面板
3. **鼠标离开** → `isFirstMenuHovering = false` → 延迟 100ms 隐藏面板
4. **点击二级菜单** → 打开 Tab → 跳转到目标页面 → 不改变面板显示状态

**关键实现**：
- `FirstSidebar.vue`：监听 `@mouseenter` 和 `@mouseleave` 设置 `isFirstMenuHovering`
- `SecondSidebar.vue`：watch `activeFirstMenu` 时检查 `isFirstMenuHovering` 才显示面板
- `app.ts`：统一管理所有导航状态

## 非菜单路径处理规范

### 什么是"非菜单路径"？

不在 `secondMenusMap` 中的路由称为非菜单路径，例如：
- `/component-preview` - 主题设置页面
- `/login` - 登录页面
- 其他不在菜单配置中的页面

### 处理原则

当访问非菜单路径时：
1. **一级菜单不选中** - `activeFirstMenu = ''`
2. **二级菜单不选中** - `activeKey = ''`
3. **展开状态清空** - `expandedKeys = []`
4. **面板不显示** - `secondSidebarHovered = false`

### 需要修改的函数

#### 1. `app.ts` - syncMenuState

```typescript
function syncMenuState(tabKey: string, tabPath: string) {
  for (const [firstKey, seconds] of Object.entries(secondMenusMap.value)) {
    for (const menu of seconds) {
      // ... 正常菜单匹配逻辑
    }
  }
  // 非菜单路径，清空状态
  activeFirstMenu.value = ''
  activeKey.value = ''
  expandedKeys.value = []
  secondSidebarHovered.value = false
}
```

#### 2. `SecondSidebar.vue` - watch(route.path)

```typescript
watch(() => route.path, (path) => {
  if (path === '/home' || path === '/') return

  const result = appStore.navigateToPath(path)
  if (result) {
    // ... 正常菜单处理
  } else {
    // 非菜单路径，清空状态
    activeKey.value = ''
    expandedKeys.value = []
  }
}, { immediate: true })
```

#### 3. `FirstSidebar.vue` - watch(route.path)

```typescript
watch(() => route.path, (path) => {
  if (path === '/home') {
    activeKey.value = 'home'
  } else if (path === '/favorites') {
    activeKey.value = 'favorites'
  } else if (appStore.activeFirstMenu) {
    activeKey.value = appStore.activeFirstMenu
  } else {
    // 非菜单路径，清空选中状态
    activeKey.value = ''
  }
}, { immediate: true })
```

#### 4. 特殊页面跳转 - handleThemeSettings

```typescript
function handleThemeSettings() {
  adminPopoverVisible.value = false
  // 打开主题设置标签页
  appStore.openTab('theme-settings', '主题设置', '/component-preview')
  // 立即清空状态
  appStore.activeFirstMenu = ''
  appStore.activeKey = ''
  appStore.expandedKeys = []
  appStore.secondSidebarHovered = false
  activeKey.value = ''
  // 跳转路由
  router.push('/component-preview')
}
```

## 动态路由

- `router/index.ts`：未知路径自动注册路由
- `router/pathComponentMap.ts`：路由到组件路径映射
- 无映射时用 BuildingView 兜底

### ⚠️ 重要：路由配置规范

**新增路由必须在两个文件中同时添加**：
1. `router/index.ts` 的 `DYNAMIC_ROUTES` 对象
2. `router/pathComponentMap.ts` 的 `getComponentPath` 中的 `map` 对象

缺少任一文件都会导致路由无法注册，页面显示"页面开发中"（BuildingView.vue）。
